
## 使用子路由处理业务的场景下，需要进行父路由局部更新
- 方案1 
	状态管理缓存父页面VM对象，mutation变更时触发父页面的更新操作
- 方案2
	状态管理增加业务点变更标识，在父页面监测到变更执行更新操作
	store.js里加代码
	```
		// 业务变更需要更新父页面的局部模块，在父页面中
		// watch:{ 'subRoutePoints.visitManger', ()=>{TODO;} }
		// 每次变更+1 ，当超过9999时，重置为0
		subRoutePoints:{
			visitManger: 0, //添加拜访户时要更新拜访户列表
		}
	```
